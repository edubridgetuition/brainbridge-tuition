System Architecture & Feature Report

Complete functional overview, API specs, database structure, and core algorithms of the Multi-Tenant Tuition ERP.

Module-Wise Feature Breakdown

The application consists of several modular systems designed for administrators, teachers, and student/parent viewports under a secure multi-tenant structure.

SuperAdmin Panel

Acts as the master licensing dashboard. Enables creating new tenants, toggling page-level visibility (students register, timetable features, analytics), configures custom receipt footprints (subheader and footer disclaimer notes), and monitors overall platform usage statistics (number of students, database size in KBs per tenant).

Timetable (Manual & AI Solver)

Provides dual options: a manual calendar grid scheduler to assign batch classes, and a local deterministic AI algorithm designed to resolve room conflicts and teacher timing overlaps instantly by matching slot availabilities.

Staff Management

Manages approval flow for newly registered teachers. Clicking a teacher's name triggers a modal detailing credentials, designation, and an interactive **Subjects Taught** editor. Supports standard checkbox listings, custom tag definitions, and a secure password reset dispatcher.

Student Performance Analytics

Aggregates test marks, homework completion efficiency, and class attendance percentages. Generates dynamic line charts of academic progress, peers rankings benchmarks, chapter-specific Gemini AI diagnostic remarks, and exports a clean printable Report Card PDF.

Homework & Submission Tracker

Allows teachers to assign study materials and assignments. Supports uploading Base64 attachment files (constrained under a strict 50KB memory limit to optimize storage). Features a submission sheet indicating Status (Submitted, Late, Not Submitted) and submission date stamps.

Fees & Collections

Tracks due dates, outstanding fees, and paid records. Features a clean search engine mapping dates, amount ranges, and student names. Generates a custom receipts pop-up and dispatches direct payment collection links via WhatsApp.

Core Algorithm: AI Timetable Scheduler

EduBridge utilizes a local conflict-resolution algorithm to generate schedule blocks while respecting multiple constraints (teacher availability, room capacities, and batch assignments).

function solveTimetable(batches, teacherAvailabilities, roomCapacities, targetDate) { let scheduleDraft = []; let teacherBookings = new Map(); // Maps "TeacherName_TimeBlock" -> boolean let roomBookings = new Map(); // Maps "RoomName_TimeBlock" -> boolean for (let batch of batches) { let timeBlock = batch.timing; // e.g. "04:00 PM - 05:00 PM" let teacher = batch.teacher_name; // Constraint 1: Check Teacher double-booking let teacherKey = `${teacher}_${timeBlock}`; if (teacherBookings.has(teacherKey)) { continue; // Skip slot to prevent teacher overlap conflict } // Constraint 2: Find available room without double-booking let allocatedRoom = null; for (let room of roomCapacities) { let roomKey = `${room.name}_${timeBlock}`; if (!roomBookings.has(roomKey)) { allocatedRoom = room.name; roomBookings.set(roomKey, true); break; } } if (allocatedRoom) { teacherBookings.set(teacherKey, true); scheduleDraft.push({ batch_id: batch.id, date: targetDate, start_time: parseStart(timeBlock), end_time: parseEnd(timeBlock), subject: batch.subject, teacher_name: teacher, room: allocatedRoom }); } } return scheduleDraft; }

Core Algorithm: Student Academic Standing Rating

A heuristic formula calculates cumulative indicators to score and rank student progress based on attendance, test marks, and homework regularity.

const calculateStudentStanding(attendanceRate, testMarksList, homeworkList, studentId) { // 1. Attendance Weight (30%) let attendanceScore = attendanceRate * 100; // 2. Test Scores Weight (50%) let studentMarks = testMarksList.filter(m => m.student_id === studentId); let avgTestPercent = studentMarks.length > 0 ? (studentMarks.reduce((sum, m) => sum + (m.marks_obtained / m.max_marks), 0) / studentMarks.length) * 100 : 0; // 3. Homework Submission Weight (20%) let studentHw = homeworkList.filter(h => h.submissions && h.submissions[studentId]); let homeworkScore = studentHw.length > 0 ? (studentHw.filter(h => h.submissions[studentId].status === 'Submitted' || h.submissions[studentId].status === 'Late').length / studentHw.length) * 100 : 0; // Cumulative Grade Calculation let totalScore = (attendanceScore * 0.3) + (avgTestPercent * 0.5) + (homeworkScore * 0.2); if (totalScore >= 90) return 'A+'; if (totalScore >= 75) return 'A'; if (totalScore >= 60) return 'B'; if (totalScore >= 45) return 'C'; return 'F'; }

Database Collections Schema (Data Dictionary)

The following data dictionaries list the exact schemas, datatypes, key constraints, and purposes of every collection stored in Firestore (scoped under the SaaS tenant path) and LocalStorage.

1. Collection: "tenants"

Stores core licensing configurations and branding parameters for each center. (Global Namespace)

Field Attribute Data Type Key Constraint Description / Mock Example
id String Document ID The unique tenant code identifier (e.g. "owner_a", "edubridge").
name String - Coaching/Tuition center name (e.g. "EduBridge Tuition").
logo_url String - Path pointing to custom loaded branding logo assets.
owner_whatsapp String - Contact mobile number used for system alerts and WhatsApp redirects.
admin_password String - Secured password credential for tuition Owner.
features Map (Object) - Boolean map containing page permission controls (e.g., { students: true, timetable: true }).
created_at ISO String - Timestamp of registration date.

2. Collection: "batches"

Defines student classroom groups under a specific subject and timing block.

Field Attribute Data Type Key Constraint Description / Mock Example
id String Document ID Generated batch key (e.g. "b1", "10th_maths").
name String - Batch display title (e.g. "10th Maths").
subject String - Academic topic/subject assigned (e.g. "Mathematics").
timing String - Scheduled duration block (e.g. "04:00 PM - 05:00 PM").
teacher_name String - Primary instructor name assigned to this batch.

3. Collection: "students"

Maintains student profiles, parent details, and active classroom links.

Field Attribute Data Type Key Constraint Description / Mock Example
id String Document ID Custom string combining name and ID (e.g. "Amit_Sharma_1001").
student_id Number Unique Numeric ID Numerical sequence identifier used for logins (e.g. 1001).
name String - Full name of the student.
mobile String - Student contact number.
parent_mobile String - Father/Mother mobile number. Used for fee reminders and auto-generated passwords.
parent_name String - Name of parent/guardian.
batch_id String Foreign Key Refers to the associated batches.id identifier.
archived Boolean - Indicates whether the student record is archived (deleted from active views).

4. Collection: "staff_accounts"

Stores credentials, profile statistics, and subjects assigned to teachers.

Field Attribute Data Type Key Constraint Description / Mock Example
id String Document ID Custom key combining name and mobile (e.g. "Aditya_7227001692").
name String - Full name of teacher.
mobile String - Primary phone number used as username credentials.
password String - Registered credentials.
subjects Array (Strings) - List of subjects this teacher is assigned to teach (e.g. ["Mathematics", "Physics"]).
subject String - Comma-separated copy of subjects array. Backwards compatible with legacy views (e.g. "Mathematics, Physics").
status String - Registration approval status: "Pending", "Approved", or "Rejected".
