Application Flowcharts

🔑 Phase 1: Launch & Authentication Flow

Shows how the application handles session checking, role-based dashboard routing, and new user sign-up approval waitlist states.

graph TD
    A["📱 App Launched"] --> B{"Session Checked?"}
    B -- "Session Valid (Auto-login)" --> C{"Determine Role"}
    B -- "No Session / Logged Out" --> D["🔐 Login Screen"]
    
    D --> E["Enter Email / Phone & Password"]
    E --> F{"Credentials Valid?"}
    F -- "Yes" --> C
    F -- "No" --> G["⚠️ Display Error (Retry)"]
    G --> D

    D -. "New User?" .-> H["📝 Registration Portal"]
    H --> I["Select Role: Teacher or Student"]
    I --> J["Enter Center Tuition Code"]
    J --> K["Submit Info & Set Password"]
    K --> L["⏳ Pending Admin Approval Screen"]
    L -. "Admin Approves" .-> D
    
    C -- "Role = Admin" --> DashboardAdmin["👑 Admin Dashboard"]
    C -- "Role = Teacher" --> DashboardTeacher["👨‍🏫 Teacher Dashboard"]
    C -- "Role = Student/Parent" --> DashboardStudent["🎓 Student/Parent Portal"]
